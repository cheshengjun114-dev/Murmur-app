# 강사 피드백 반영 결정 사항

## 1. 관리자도 작성자를 알 수 없는가?

아닙니다. Murmur의 익명성은 "사용자 간 익명"입니다.

관리자는 신고 처리, 규정 위반 글 제재, 악성 이용자 관리가 필요하기 때문에 작성자의 내부 사용자 ID를 확인할 수 있어야 합니다. 다만 일반 사용자 화면에는 이메일, 닉네임, 사용자 ID를 절대 노출하지 않고 익명 번호만 보여줍니다.

운영 정책:

- 일반 사용자: 작성자 계정 정보 확인 불가
- 관리자: 신고/제재 목적에 한해 작성자 ID 확인 가능
- 서버 DB: 게시글, 댓글, 대댓글에 실제 작성자 `userId` 저장
- 관리자 조회 기능: 추후 로그를 남겨 오남용 방지

## 2. 익명 번호 정책

익명 번호는 "게시글 상세 화면 단위"로 유지합니다.

결정:

- 게시글 작성자: 항상 `익명1(글쓴이)`
- 첫 번째 댓글 작성자: `익명2`
- 두 번째로 등장한 다른 댓글 작성자: `익명3`
- 같은 사용자가 같은 게시글 안에서 다시 댓글/대댓글 작성: 기존 익명 번호 유지
- 게시글 작성자가 댓글/대댓글 작성: `익명1(글쓴이)` 유지
- 다른 게시글에서는 익명 번호를 새로 부여

예시:

```text
게시글: 익명1(글쓴이)
댓글 A: 익명2
댓글 B: 익명3
글쓴이 답글: 익명1(글쓴이)
댓글 A가 다시 답글: 익명2
```

이 방식이면 `익명5`가 글쓴이인지 댓글러인지 헷갈리는 문제가 사라집니다.

## 3. 신고 정책

신고 정책은 서비스 운영의 핵심이므로 MVP 단계부터 명확히 고정합니다.

기본 정책:

- 로그인 사용자만 신고 가능
- 자기 글/댓글/대댓글 신고 불가
- 같은 사용자는 같은 대상에 대해 1회만 신고 가능
- 게시글 신고 5회 이상: 자동 숨김
- 댓글/대댓글 신고 3회 이상: 자동 숨김
- 관리자는 숨김 처리된 콘텐츠를 해제/유지/삭제할 수 있음
- 자동 숨김만으로 즉시 페널티를 부여하지 않음
- 관리자가 검토 후 경고, 기간 정지, 영구 정지 결정

숨김 해제 정책:

- 관리자가 숨김 해제한 콘텐츠는 `adminUnhidden=true` 같은 상태값을 둡니다.
- 같은 신고 누적으로 즉시 다시 자동 숨김 처리되지 않도록 막습니다.
- 새 신고가 추가로 들어오면 다시 검토 대상으로 올릴 수 있습니다.

## 4. Redis 사용 이유

Redis는 "빠른 임시 저장소" 역할로 사용합니다.

MVP에서 바로 쓰는 용도:

- Refresh Token 저장
- 로그아웃 시 Refresh Token 삭제
- 토큰 만료 시간 TTL 자동 관리
- 강제 로그아웃 또는 계정 정지 시 토큰 무효화

추후 확장 가능한 용도:

- 인기글 점수 캐싱
- 조회수 증가량 임시 저장
- 신고 카운트 캐싱
- 동일 사용자의 반복 요청 제한

면접 답변 예시:

> MySQL은 영구 데이터를 저장하고, Redis는 만료 시간이 중요한 인증 정보나 자주 바뀌는 카운트 데이터를 빠르게 처리하기 위해 사용합니다. 특히 Refresh Token을 Redis에 저장하면 로그아웃과 토큰 만료 처리를 TTL로 깔끔하게 관리할 수 있습니다.

## 5. 익명인데 회원가입을 받는 이유

익명은 "화면에 내 신원이 공개되지 않는다"는 의미이지, 서버가 아무도 구분하지 않는다는 뜻은 아닙니다.

회원가입이 필요한 이유:

- 내가 쓴 글 수정/삭제 권한 확인
- 댓글/대댓글 작성자 검증
- 중복 신고 방지
- 자기 글 신고 차단
- 악성 사용자 제재
- 북마크, 마이페이지 제공

## 6. 같은 사람이 쓴 글 추적 가능 여부

일반 사용자는 추적할 수 없습니다.

- 공개 화면: 게시글마다 익명 번호가 새로 부여됨
- 일반 사용자: 다른 게시글의 작성자가 같은 사람인지 알 수 없음
- 관리자: 신고/제재 목적에 한해 특정 계정의 작성 이력 확인 가능

## 7. 익명 글 수정/삭제 방식

서버는 작성자를 알고 있고, 화면에는 숨깁니다.

처리 흐름:

```text
1. 사용자가 게시글 수정/삭제 요청
2. JWT에서 로그인 사용자 ID 추출
3. DB에서 게시글 작성자 userId 조회
4. 로그인 사용자 ID와 작성자 ID 비교
5. 같으면 허용, 다르면 403 Forbidden
```

이 구조로 익명성과 권한 검증을 동시에 만족시킵니다.
