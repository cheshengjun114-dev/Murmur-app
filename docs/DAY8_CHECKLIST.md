## 익명 번호 정책

- 게시글 작성자는 `익명1(글쓴이)`입니다.
- 댓글/대댓글 작성자는 게시글 안에서 처음 등장한 순서대로 `익명2`, `익명3`처럼 부여됩니다.
- 익명 번호는 `anonymous_mappings` 테이블에 저장됩니다.
- 같은 사용자는 같은 게시글 안에서 항상 같은 익명 번호를 유지합니다.
- 다른 게시글에서는 익명 번호가 새로 부여됩니다.

## 신고 API

### 게시글 신고

```http
POST /api/posts/{postId}/report
Authorization: Bearer {accessToken}
Content-Type: application/json
```

```json
{
  "reason": "비방/욕설이 포함되어 있습니다."
}
```

응답:

```json
{
  "success": true,
  "message": "요청이 성공했습니다.",
  "data": {
    "postId": 1,
    "reportCount": 1,
    "blinded": false
  }
}
```

## 오류 정책

- 본인 게시글 신고: `400 Bad Request`
- 같은 사용자의 중복 신고: `409 Conflict`
- 없는 게시글 신고: `404 Not Found`
